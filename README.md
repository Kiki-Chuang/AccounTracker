# AccounTracker | 銀行自動化核帳工具 (CTBC 專用版)

這是一個專為會計與財務人員設計的輕量化 Web 工具，旨在解決「銀行網銀明細」與「公司內部 Excel」格式不一、人工核對耗時且易出錯的痛點。

## 💡 開發動機
在會計實務中，經常需要核對大量的匯款待放款明細。由於銀行後台資料與公司內部 ERP 匯出的 Excel 欄位格式截然不同，傳統的人工「肉眼對帳」不僅效率低下，更存在遺漏風險。

本專案透過 JavaScript 實作**雙軌比對演算法**，將原本需耗時 30 分鐘的核對工作縮短至 **3 秒內**完成。

## 🚀 技術亮點

### 1. 雙軌精準匹配機制
- **優先權重比對**：程式會優先尋找「備註/附言」完全符合的項目。
- **Fallback 備援機制**：若備註不符，則啟動「金額匹配」並標記為建議，確保不會漏掉任何可能的對應筆數。

### 2. 瀏覽器端即時解析 (Zero-Server Architecture)
- **資安友善**：所有資料僅在使用者瀏覽器本地端運算，**絕不回傳至任何伺服器**。這對於處理財務個資、帳號、金額等敏感資訊至關重要。
- **TSV 解析技術**：支援從 Excel 直接 `Ctrl+C` / `Ctrl+V` 貼上，自動解析 Tab-Separated Values 格式。

### 3. 資料歸一化 (Data Normalization)
- 自動將「全形數字/英文字母」轉換為「半形」，排除千分位逗號、空格與特殊符號對比對結果的干擾。
- 針對銀行帳號自動進行補零 (Padding) 與格式統一。

## 🛠️ 輔助工具：網銀資料擷取腳本 (`ctbc-scraper.js`)
由於中信網銀單筆匯款頁面未提供 CSV 下載功能，本專案另外開發了專用抓取腳本。
- **核心邏輯**：利用 JavaScript 深入 DOM 節點，精準分離「處理序號」與「匯款金額」。
- **自動化下載**：一鍵產生帶有 BOM (Byte Order Mark) 的 CSV 檔案，完美解決 Excel 開啟亂碼問題。

## 🛠️ 使用技術
- **Vanilla JavaScript (ES6+)**: 核心比對邏輯、DOM 操作。
- **Tailwind CSS**: 響應式 UI 設計。
- **PapaParse**: 處理大數據量 CSV 的高效能解析庫。

## 📖 使用說明
1. **取得銀行資料**：在網銀頁面執行 `ctbc-scraper.js` 下載 A 區 CSV。
2. **上傳 A 區**：將下載的 CSV 匯入網頁。
3. **貼上 B 區**：從你的工作 Excel 複製對應範圍，直接貼入 B 區文字方塊。
4. **自動比對**：點擊「開始自動比對」，系統將以綠、黃、紅三色即時標示核帳結果。
